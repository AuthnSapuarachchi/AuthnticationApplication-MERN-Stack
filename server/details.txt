This is a MERN authentication app with email-based verification, password reset, role-based dashboards, and optional two-factor authentication. The frontend is a Vite React app, and the backend is an Express + MongoDB API with JWT cookies and Nodemailer-based email delivery. The main wiring is in main.jsx, AppContext.jsx, and server.js.

Frontend-wise, the app has these screens: Home, Login/Signup, Reset Password, Email Verification, Settings, 2FA Setup, 2FA Verify, User Dashboard, and Admin Dashboard. The home page combines a top nav and a hero section in Home.jsx, NavBar.jsx, and Header.jsx. The auth flow is centered in Login.jsx: registration and login share the same component, signup is routed through the same page, and login can branch into 2FA verification if the account has 2FA enabled. Password reset and email verification are separate OTP flows in ResetPassword.jsx and EmailVerify.jsx. Account security management lives in Settings.jsx, while TwoFactorSetup.jsx and TwoFactorVerify.jsx handle TOTP setup and login-time verification. The user and admin views are in UserDashboard.jsx and AdminDashboard.jsx.

Backend-wise, the API is split into auth, user, and 2FA routers: authRoutes.js, userRoutes.js, and twoFactorRoutes.js. The auth controller in authController.js handles registration, login, logout, token refresh, email verification OTPs, password reset OTPs, and test endpoints. The user controller in userController.js serves current user data and builds the dashboard payloads for standard users, moderators, and admins. The 2FA controller in twoFactorController.js generates QR codes and backup codes, verifies TOTP codes, accepts backup codes during login, disables 2FA, reports 2FA status, and regenerates backup codes. Authentication is enforced by userAuth.js, which reads the JWT cookie and injects user context into the request, and role checks are done by roleAuth.js.

The data model is in usermodel.js. It stores name, email, password, verification OTPs, reset OTPs, role, refresh tokens, 2FA secret, 2FA enabled flag, and backup codes. There is also a simpler legacy file at User.js, but it is not the one the app actually imports. Email delivery is configured in nodemailer.js, with fallback/alternate configs in nodemailer-gmail.js, nodemailer-alternative.js, and email HTML templates in emailTemplates.js and emailTemplates.js. MongoDB connection logic is in mongodb.js.
