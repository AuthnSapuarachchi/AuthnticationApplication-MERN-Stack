## 🔍 Edge Cases to Test

### Edge Case 1: Expired Code
**Test:** Wait 30+ seconds, enter old code
**Expected:** Error toast "Invalid verification code"

### Edge Case 2: Wrong Code
**Test:** Enter random 6 digits
**Expected:** Error toast "Invalid verification code"

### Edge Case 3: Incomplete Code
**Test:** Enter only 4 digits, click Verify
**Expected:** Button disabled (can't click)

### Edge Case 4: Used Backup Code
**Test:** Try to use same backup code twice

## 🐛 Common Issues & Solutions

### Issue: "QR code not scanning"
**Solutions:**
- Increase screen brightness
- Use manual entry code instead
- Try different authenticator app
- Check if QR code image loaded properly

### Issue: "Invalid verification code" every time
**Solutions:**
- Check phone time is synced (Settings → Date & Time → Auto)
- Server time might be off - sync server clock
- Try code that just appeared (not about to expire)
- Window tolerance is ±2 steps (60 seconds total)

### Issue: "Backup codes not downloading"
**Solutions:**
- Check browser's download settings
- Allow downloads from localhost
- Manually copy codes to text file
- Check browser console for errors

### Issue: "Can't disable 2FA - wrong password"
**Solutions:**
- Make sure Caps Lock is off
- Copy-paste password to avoid typos
- Reset password if forgotten
- Admin can disable 2FA from database directly

### Issue: "Lost phone, no backup codes"
**Solutions:**
- Admin recovery needed
- Manually update database:
  ```javascript
  db.users.updateOne(
    { email: "user@example.com" },
    { 
      $set: { 
        twoFactorEnabled: false,
        twoFactorSecret: "",
        backupCodes: []
      }
    }
  )
  ```

## 🎯 Production Deployment Checklist

Before deploying 2FA to production:

- [ ] Test on staging environment first
- [ ] Enable HTTPS (required for secure cookies)
- [ ] Set `NODE_ENV=production` in .env
- [ ] Update CORS allowed origins
- [ ] Test from different devices
- [ ] Test from different networks
- [ ] Add rate limiting to prevent brute force
- [ ] Monitor 2FA verification failures
- [ ] Document recovery process for users
- [ ] Train support team on 2FA issues
- [ ] Add email notification when 2FA is enabled/disabled
- [ ] Consider adding SMS backup option
- [ ] Create admin dashboard to view 2FA stats
- [ ] Backup user data before deployment
